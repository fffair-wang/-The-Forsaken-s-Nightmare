# 遗忘者之噩梦 — 开发文档

> 版本：v1.0（最终版）
> 技术栈：纯 HTML / CSS / JavaScript（单文件，无外部依赖）
> 运行方式：直接在浏览器打开 HTML 文件（推荐通过 HTTP 服务器访问以避免 CORS 限制）
> 文件：`遗忘者之噩梦_嵌图版.html`（~5255 行，~7.5 MB，含嵌入式场景图片）

---

## 文件结构

```
遗忘者之噩梦_嵌图版.html  — 全部代码（HTML + CSS + JS + 嵌入式 base64 场景图片），约 5255 行
bgm/                       — BGM 音频文件夹（13 首 MP3，需与 HTML 同目录）
```

## 当前已实现功能

### 1. 页面系统

| 页面 | 说明 |
|---|---|
| **TITLE（标题页）** | 游戏标题「遗忘者之噩梦」，四个按钮：开始新游戏 / 读取存档 / 结局展览馆 / 设置 |
| **WARNING（游戏提醒）** | 三条游戏须知，确认后进入开场动画 |
| **LOAD（标题页读档）** | 3 个可选择存档位 + 右下角「加载」「删除」按钮 |
| **CHAPTER INTRO（章节开场）** | 纯黑画面，白字花体依次展现章节标题与副标题，约 5 秒后自动进入游戏；支持中英双语 |
| **GAME（游戏主页面）** | 左 50% 场景图片区（含嵌入式图片） + 右 50% 文字区 |
| **ENDING（结局页面）** | 纯黑背景，白字动画渐显（标题 0.2s → 描述 1.2s → 按钮 2.4s），三个按钮：返回标题 / 读取存档 / 结局展览馆 |
| **GALLERY（结局展览馆）** | 标题页/结局页入口，单列滚动网格，锁定结局显示标题 + 「？？？」；解锁结局显示描述 + 解锁条件 |
| **EASTER EGG（彩蛋页）** | 解锁全部 16 个结局后首次访问画廊时触发，显示 4 行祝贺语 + 点击继续进入画廊 |

### 2. 游戏交互

| 功能 | 说明 |
|---|---|
| **打字机效果** | 逐字渲染剧情文字，速度可在设置中调节（10~120ms/字） |
| **跳过打字** | 点击文字区域 / Space / Enter 可立即显示完整文字 |
| **点击以继续** | 无分支场景打字完成后，底部脉冲提示，点击进入下一场景 |
| **分支选项** | 有分支场景打字完成后，显示选项按钮，点击触发对应分支；支持中英双语选项文本 |
| **调查系统** | 多选项搜索模式，逐项查看后可点击「继续」推进剧情，每项查看后推入历史记录 |
| **逃脱小游戏** | Canvas 覆盖左半场景，利爪痕迹（红/黄/蓝三色），每轮显示数秒后提问颜色数量，4 选 1 作答；轮数根据场景动态设置（3~7 轮不等）；首遇弹教程（localStorage `nightmare_escape_tutorial`） |
| **射击小游戏** | Canvas 覆盖左半场景，子弹持续生成（每发存活 1.5s，最多 3 发同屏），点击变红即命中；子弹数根据场景动态设置（12~24 发不等）；首遇弹教程（localStorage `nightmare_combat_tutorial`） |
| **场景图片系统** | 每个场景自动加载对应的嵌入式 base64 图片，300ms 淡入过渡；图片路径映射由 `SCENE_IMAGES` 定义，数据由 `IMAGE_DATA` 提供 |
| **剧情结束提示** | 当前分支剧情结束时显示「— 未完待续 —」 |

### 3. 存档 / 读档 / 命名 / 删除

| 功能 | 触发入口 | 流程 |
|---|---|---|
| **存档** | 游戏中顶部导航栏「存档」按钮 | 弹窗 → 选存档位 → 空位→命名 / 有数据→覆盖确认→命名 → 保存 |
| **读档（游戏中）** | 游戏中顶部导航栏「读档」按钮 | 弹窗 → 选存档位 → [加载] → 二次确认 → 恢复场景 |
| **读档（标题页）** | 标题页「读取存档」按钮 | 全屏页 → 选存档位 → [加载] → 直接进入游戏并恢复场景 |
| **删除存档** | 标题页或游戏内读档页，选中后点 [删除] | 确认弹窗 → 确认→删除 / 取消→返回 |

**存档命名规则：**
- 默认名按已有存档数量生成：存档一 / 存档二 / 存档三
- 命名输入框最长 20 字，按 Enter 可快速确认

**存档数据（localStorage 持久化）：**
- `name` — 用户命名的存档名称
- `sceneId` — 当前场景 ID
- `sceneText` — 预览文字（前 100 字）
- `hasRemembered` — 是否记得战场记忆（布尔）
- `hasEavesdropped` — 是否偷听过护士长办公室对话（布尔）
- `hasInvestigatedOffice` — 是否调查过院长办公室（布尔）
- `hasSurvivedBasement` — 是否从地下室逃脱（布尔）
- `hasReleasedSubjects` — 是否释放试验体（布尔）
- `timestamp` — 存档时间戳

### 4. 设置系统

| 设置项 | 范围 | 默认值 |
|---|---|---|
| 文字速度 | 10~120ms/字，步长 5 | 50ms |
| BGM 音量 | 0~100 | 70 |
| 音效音量 | 0~100 | 70 |
| 语言 | zh / en | zh |

设置项通过 `localStorage` 持久化保存。

### 4a. 语言切换系统（中英双语）

完整的中英双语支持，全局 UI 和剧情文本均覆盖：

| 组件 | 说明 | 位置 |
|---|---|---|
| **`UI_STRINGS`** | 约 60+ 组 UI 字符串（按钮/弹窗/提示/设置等），zh + en 成对定义 | 线 1583+ |
| **`data-i18n` 属性** | 所有 UI 元素通过该属性关联 `UI_STRINGS` 键值 | 线 1317~1541 |
| **`applyLanguage()`** | 动态更新所有 `[data-i18n]` 元素文本，切换 `<html>` 的 `lang-en` 类 | 线 1718+ |
| **语言选择器** | 设置页下拉菜单，即时切换（中文 / English） | 线 1429~1431 |
| **`STORY_DATA_EN`** | 全量剧情英文翻译，通过 `getStoryData()` 合并覆盖 | 线 3654+ |
| **`ENDING_NAMES_EN`** | 16 个结局名的英文映射 | 线 3635+ |
| **`CHAPTER_INTROS`** | 三章开场副标题中英双版 | 线 1788+ |
| **条件文本双语** | `【E1】` `【E2】` `【E3】` 标记的英文替换文本 | 线 4283+ |

### 4b. 结局图鉴（Gallery）系统

- `ENDINGS_DATA` 数组定义 16 个结局（id, desc, condition），id 对应场景中 `isEnding` 值
- `unlockEnding(id)` 在 `showEnding()` 中自动调用，通过 localStorage（`nightmare_unlocks` 键）持久化
- 画廊渲染时根据解锁状态显示锁定/解锁样式：
  - 锁定：显示结局标题 + 「？？？」灰暗样式
  - 解锁：显示结局标题 + 描述文字 + 解锁条件
- 画廊可从标题页或结局页进入，通过 `endingGalleryMode` 标志控制 ESC/返回按钮跳回来源页
- 英文模式下显示 `ENDINGS_DATA.descEn` 中的英文描述

### 4c. 彩蛋系统（Easter Egg）

- **触发条件**：所有 16 个结局均已解锁 + 首次访问画廊
- **表现**：显示 4 行祝贺语（章节开场风格），2 秒后出现「点击继续」按钮，进入画廊
- **持久化**：`localStorage.nightmare_easter_seen` 标记是否已看过（仅触发一次）
- **文本**：中英文各 4 行，存放于 `UI_STRINGS` 的 `zh.easterLine1~4` / `en.easterLine1~4`

### 4d. 剧情条件变量系统

| 变量 | 作用 | 设置场景 |
|---|---|---|
| `hasRemembered` | 是否记得战场记忆，影响询问病情分支的文本走向 | `prologue_2` 选择「记得」/「不记得」 |
| `hasEavesdropped` | 是否偷听过走廊对话，影响后续 Part 2 部分场景文本 | `prologue_6_eavesdrop` 选择「再尝试听一听」 |
| `hasInvestigatedOffice` | 是否调查过院长办公室（密码破解成功），影响 Part 2 分支走向 | `prologue_14_office_correct` 选择「查看邮箱」 |
| `hasSurvivedBasement` | 是否从地下室逃脱（逃脱小游戏成功），影响 Part 2 分支走向 | 逃脱小游戏正确回答后自动设置 |
| `hasReleasedSubjects` | 是否释放试验体，影响 Part 3 失败结局走向 | `part3_inject_2` / `part3_refuse_2` 选择「释放试验体引发混乱」 |

- 变量随存档保存/读取，新建游戏时重置
- 文本中使用 `【E1】` `【E2】` `【E3】` 标记条件插入点，`resolveSceneText()` 在渲染时根据 `hasEavesdropped` 自动替换（支持中英双语替换文本）

### 5. 场景图片系统

基于嵌入式 base64 的场景图片系统，每个场景自动配图：

**组件：**
- **`SCENE_IMAGES`**（线 ~4329）— 场景 ID → 图片文件名映射，覆盖全部 50+ 个需要配图的场景
- **`IMAGE_DATA`**（线 ~4330）— 图片文件名 → base64 数据映射，52 张图片全部以 data URI 形式嵌入 HTML
- **`updateSceneImage(sceneId)`**（线 ~4334）— 从 `SCENE_IMAGES` 查文件名，从 `IMAGE_DATA` 取 base64 数据，设入 `<img>` 元素，带 300ms 淡入过渡
- **`lastSceneImage` 缓存**（线 ~4333）— 防止同一图片重复加载

**HTML / CSS：**
- `<img id="scene-image">`（线 ~1495）— 位于 `.game-left` 内
- `#scene-image` 初始 `opacity: 0`，`.visible` 类切换至 `opacity: 1`，0.6s CSS transition

**添加新图片：** 在 `IMAGE_DATA` 中添加 base64 编码的图片数据，在 `SCENE_IMAGES` 中添加场景 ID → 文件名映射即可。

### 6. 音效系统

基于 Web Audio API 生成的占位音效，无需外部音频文件：
- 按钮悬停：轻响
- 按钮点击：短促声
- 确认操作：双音效
- 音量受设置中「音效音量」控制

**机制说明：**
- Ending 场景（`isEnding`）跳过 `updateBGM`，BGM 继承自上一场景，在 `showEnding()` 中于按钮出现时（2.4s）执行 3s fadeOut
- `crossfadeBGM(newSrc, 500)` — 500ms 重叠淡入淡出
- `crossfadeBGM(null, 500)` → `fadeOutBGM(500)` 淡出至静音
- `playBGM()` 自动去重（相同 src 不重复播放）
- `loop=true` + `timeupdate` 事件检测循环点 → 每循环 300ms 音量淡入
- 音量与 `settings.bgmVolume` 同步

### 6b. 历史记录（Review Panel）

- 左侧覆盖层，覆盖左 50% 画面
- 显示最近 2 段场景文本（最早在上），游戏推进时自动追加
- 游戏中顶部栏「回顾」按钮打开，ESC 关闭

### 6c. 小游戏安全清理

`stopAllMinigames()` 函数（线 ~4305）在 `loadScene()` 开头调用，清理逃脱/射击小游戏的定时器、事件监听与覆盖层，防止读档或场景切换时小游戏残留。

## 扩展指南

### 添加新场景

在 `storyData` 对象中添加：

```javascript
// 线性场景
scene_id: {
  text: '场景文字。\n\n换行用 \\n\\n。',
  next: 'next_scene_id'
}

// 分支场景
scene_id: {
  text: '场景文字。',
  choices: [
    { text: '选项 A', next: 'scene_a' },
    { text: '选项 B', next: 'scene_b' }
  ]
}

// 调查（搜索）场景
scene_id: {
  text: '场景文字。',
  investigation: [
    { label: '查看某物', text: '调查反馈文本……' },
    { label: '查看另一处', text: '调查反馈文本……' }
  ],
  next: 'next_scene_id'   // 所有调查项看完后自动跳转
}

// 结局场景
scene_id: {
  text: '场景文字。',
  next: null,
  isEnding: '结局名称',
  endingDesc: '结局描述文字'
}

// 章节过渡场景
scene_id: {
  text: '场景文字。',
  part2Intro: true | false
}
```

- `next: null` 表示剧情终点，显示「未完待续」
- `next: null` + `part2Intro: true` 表示进入下一章开场动画（用于章节间过渡）
- `isEnding: '结局名称'` + `endingDesc: '描述文字'` 表示结局场景，自动解锁结局图鉴
- `part2Intro: true` 标记场景推进时先播放 Part 2 章节过渡动画
- `part3Intro: true` 标记场景推进时先播放 Part 3 章节过渡动画
- `investigation: [...]` 定义调查项目列表，每项有 `label`（选项按钮文字）和 `text`（查看后显示的文字）
- 场景分支嵌套深度不限

### 添加英文文本

```javascript
// 在 STORY_DATA_EN 中添加对应场景 ID 的英文覆盖：
STORY_DATA_EN['scene_id'] = {
  text: 'English text.',
  choices: [
    { text: 'Option A', next: 'scene_a' }
  ]
}
```

`getStoryData()` 在语言为 `'en'` 时自动合并 `STORY_DATA_EN` 中的覆盖字段。

### 添加场景图片

1. 将图片 base64 编码后加入 `IMAGE_DATA` 对象：
   ```javascript
   "filename.jpg": "data:image/jpeg;base64,/9j/4AAQ..."
   ```
2. 在 `SCENE_IMAGES` 中添加场景 ID → 文件名映射：
   ```javascript
   "scene_id": "filename.jpg"
   ```
3. 系统自动在场景加载时读取并显示对应图片（300ms 淡入过渡）。

### 条件变量处理

在 `onTypeComplete()` 中的 `choice` 点击回调里，根据 `currentSceneId` 设置全局变量：

```javascript
if (currentSceneId === 'prologue_2') {
  hasRemembered = (c.next === 'prologue_3a');
}
if (currentSceneId === 'prologue_6_eavesdrop') {
  hasEavesdropped = true;
}
```

变量在存档时保存，读档时恢复。

### `getStoryData()` 与 `advanceScene()` 流程

`getStoryData()` 函数返回带英文覆盖的数据：
```javascript
function getStoryData() {
  if (settings.language !== 'en') return storyData;
  // 合并 STORY_DATA_EN 覆盖
}
```

`advanceScene(scene)` 流程：
- 若 `scene.part2Intro` 为 true，触发 `showChapterIntro(2, lines, callback)` 播放 Part 2 开场动画后再跳转
- 若 `scene.part3Intro` 为 true，触发 `showChapterIntro(3, lines, callback)` 播放 Part 3 开场动画后再跳转
- 否则直接调用 `loadScene(scene.next)`

章节开场支持中英双语，根据 `settings.language` 自动选择对应文本。

### 条件文本系统

使用 `【E1】` `【E2】` `【E3】` 标记根据 `hasEavesdropped` 变量条件显示的文字：

- `hasEavesdropped = true` → 标记替换为对应的偷听参考文本（支持中英双语）
- `hasEavesdropped = false` → 标记被移除（空字符串）

在 `resolveSceneText(scene)` 中自动处理，可用于历史记录回显和正在渲染的文本。

### 章节开场（Chapter Intro）

调用方式：
```javascript
showChapterIntro(partNumber, callback);
```

- `partNumber` — 显示的 Part 编号（1/2/3）
- `callback` — 动画结束后执行的回调

副标题文本根据 `settings.language` 自动选择：
- Part 1（ZH）：久远飘渺的人啊；可曾还记得你的名？
- Part 1（EN）：O distant, ethereal soul; Can you still remember your name?
- Part 2（ZH）：看，血罪的种子将要发芽；听，告喻的钟声已然敲响
- Part 2（EN）：See, the seeds of blood-guilt shall sprout; Hark, the bell of revelation tolls.
- Part 3（ZH）：汝以罪人之血淬洗剑锋；你的剑必折，锋刃必将你划伤
- Part 3（EN）：You have whetted your blade on sinners' blood; Your sword shall break — its edge shall cut you deep.

### 逃脱小游戏（Escape Game）

```javascript
// 场景触发
scene_id: {
  text: '场景文字，之后进入逃脱游戏。',
  escapeGame: {
    rounds: 5,       // 轮数（3~7 轮不等）
    nextSuccess: 'scene_win_id',
    nextFail: 'scene_lose_id'
  }
}
```

通过 Canvas 绘制利爪划痕（红/黄/蓝三色），每轮显示数秒后隐藏，提问该轮出现的颜色数量。轮数根据场景动态配置（触发场景默认 4 轮，释放试验体 5/7 轮，闪避 3~5 轮不等）。

### 射击小游戏（Combat Game）

```javascript
scene_id: {
  text: '场景文字，之后进入射击游戏。',
  combatGame: {
    totalBullets: 12,        // 总生成子弹数（12~24 不等）
    displayMs: 1500,         // 每发子弹存活时长（ms）
    spawnInterval: 700,      // 生成间隔（ms）
    maxConcurrent: 3,        // 最大同屏子弹数
    nextSuccess: 'scene_win',
    nextFail: 'scene_lose'
  }
}
```

- Canvas 绘制传统手枪子弹轮廓（弹尖 + 柱体 + 槽线 + 底缘），白色→红色（命中）+ 发光
- 持续生成，最大同屏 3 发，每发 1.5s 自动消失
- 总子弹数根据场景动态设置（12/15/18/20/24）
- 全点击变红（命中）则胜利，否则失败
- 首次遇战弹出教程，通过 `localStorage.nightmare_combat_tutorial = true` 记录

### 添加 BGM / 音效

基于 `<audio>` 标签的 BGM 系统，支持交叉淡入淡出、循环淡入、音量同步。BGM 文件存放于 `bgm/` 子目录。

**添加新 BGM 轨：**
1. 在 `BGM_TRACKS` 中添加新键值：
   ```javascript
   newKey: BGM_BASE + 'filename.mp3'
   ```
2. 在 `updateBGM()` 的 else-if 链中添加场景匹配。

**音效系统**基于 Web Audio API 生成占位音效（按钮悬停/点击/确认），无需外部文件，音量受 `settings.sfxVolume` 控制。

### 小游戏安全清理

`stopAllMinigames()` 在 `loadScene()` 开头自动调用，清理逃脫/射击小游戏的定时器与事件监听。新增小游戏类型时，应在该函数中添加对应的清理逻辑。
